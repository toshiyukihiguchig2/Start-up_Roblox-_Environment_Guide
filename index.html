Roblox Development: The Essential Setup Guide

最終更新日：2026/03/05

1. プロジェクトの目的

本プロジェクトでは、Roblox上で動作するアクションゲームの開発を効率化するため、以下の開発フローを採用します。

Rojo TestEZ GitHub Actions
  • Rojo：外部エディタ（VS Code）とRoblox Studioの同期。
  • TestEZ：ユニットテストによる品質管理。
  • GitHub Actions：push時の自動ビルド・テスト検証（CI）。

2. GitHubの準備

2.1. アカウント作成とGitインストール

  1. GitHubアカウント作成：GitHub.com でアカウントを作成。
  2. Gitインストール：Git for Windows をインストール。
  3. 初期設定：PowerShellを開き、以下を実行。
powershellgit config --global user.name "YourName"
git config --global user.email "your-email@example.com"

2.2. リポジトリの作成

  1. GitHubで [New repository] を作成。
  2. Repository name：roblox-project-name（任意）
  3. Public / Private：プロジェクトに合わせて選択。
  4. Initialize：「Add a README file」にチェック、「.gitignore」は Roblox を選択。
  5. 作成後、緑色の [<> Code] ボタンからリポジトリのURL（HTTPS）をコピー。

3. 開発環境のセットアップ

3.1. リポジトリのクローン

powershell# 作業用フォルダへ移動（例）
cd C:\Users\<PCユーザー名>\source\repos
# クローン実行
git clone https://github.com/<GitHubユーザー名>/roblox-project-name.git
cd roblox-project-name

3.2. 必須ツールのインストール

1. Rojo & Wally

1.1. Rojo

  • Rojo Releases からWindows用zipをダウンロード。
  • rojo.exe を任意のフォルダ（例：C:\Users\<PCユーザー名>\AppData\Local\Programs\DevTools）に配置。
  • 上記フォルダを 環境変数 (PATH) に追加。

1.2. Wally

  • Wally Releases からWindows用zipをダウンロード。
  • wally.exe を任意のフォルダ（例：C:\Users\<PCユーザー名>\AppData\Local\Programs\DevTools）に配置。
  • 上記フォルダを 環境変数 (PATH) に追加。

2. VS Code 拡張機能

  • Rojo
  • Luau Language Server

4. 開発フロー

4.1. サーバー起動と同期

  1. VS Codeでプロジェクトを開き、ターミナルで実行：
powershellrojo serve
  1. Roblox Studioを開き、Rojoプラグインから「Connect」を押下。
src/ 内のコードが Studio の ReplicatedStorage 等へ同期されます。

4.2. テストの実行 (TestEZ)

  1. 依存関係の導入：wally install を実行し Packages/ を生成。
  2. テストコード作成：src/tests/ 内に .spec.lua ファイルを作成。
  3. 実行：Studioの「プレイ (F5)」を押すと、TestRunner スクリプトにより自動でテスト結果が出力されます。

5. CI (GitHub Actions) の設定

リポジトリの .github/workflows/test.yml に以下を配置することで、pushのたびに自動テストが走ります。

yamlname: Test
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Foreman (Tools)
        run: |
          curl -L https://github.com/Roblox/foreman/releases/latest/download/foreman-linux-x86_64.zip -o foreman.zip
          unzip foreman.zip
          chmod +x foreman
          sudo mv foreman /usr/local/bin/
      - name: Install Wally & Rojo
        run: foreman install
      - name: Add Path
        run: echo "$HOME/.foreman/bin" >> $GITHUB_PATH
      - name: Build & Verify
        run: rojo build test.project.json -o test.rbxl

6. 運用ルールと注意点

.gitignore の徹底

以下のファイルは 絶対にGitに含めない でください（リポジトリの肥大化や競合を防ぐため）。
  • .rbxl、.rbxm（Robloxバイナリファイル）
  • Packages/（Wallyで管理するため）
  • .vscode/（個人設定）

基本的なGitコマンド

操作 コマンド
変更の保存 git add -A → git commit -m "feat: 〇〇の実装"
サーバーへ送信 git push
最新情報の取得 git pull
